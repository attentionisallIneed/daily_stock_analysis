# 金融策略逻辑与优化路线

本文档沉淀当前项目在金融策略层面的实现逻辑、优势、不足与后续优化方向，作为后续迭代策略模块、回测模块、风控模块和 LLM 决策提示词的依据。

> 本项目输出内容用于投资研究和自选股复盘辅助，不构成投资建议或自动交易指令。

## 1. 当前系统定位

当前项目是一个 A 股自选股日度决策辅助系统，而不是完整自动交易系统。

核心目标：

1. 对配置的自选股做日线级别分析。
2. 获取历史行情、实时行情、筹码分布、大盘环境与新闻情报。
3. 用规则层先形成可解释的趋势、量能、买点和风险判断。
4. 将结构化上下文交给 OpenAI 兼容 LLM 生成决策仪表盘。
5. 通过通知渠道、报告文件或飞书文档输出每日复盘结果。

整体策略风格：

- 趋势跟随。
- 严格不追高。
- 趋势中回踩买入。
- 大盘环境过滤。
- 筹码与消息面辅助排雷。
- LLM 负责解释和组织报告，硬交易纪律由规则层兜底。

## 2. 当前分析链路

主流程由 `StockAnalysisPipeline` 编排：

1. 从配置读取股票池。
2. 通过数据源获取 250 日左右日线行情并写入 SQLite。
3. 获取实时行情，包括现价、量比、换手率、PE、PB、市值等。
4. 获取筹码分布，包括获利比例、平均成本、筹码集中度。
5. 调用 `StockTrendAnalyzer` 做规则化趋势分析。
6. 调用搜索服务获取最新消息、风险排查、业绩预期。
7. 获取大盘环境并形成市场风控上下文。
8. 将行情、趋势、筹码、新闻、大盘环境合并为增强上下文。
9. 调用 LLM 生成 JSON 决策仪表盘。
10. 对 LLM 结果执行硬规则修正。
11. 生成报告并推送。

关键模块：

- `main.py`：主流程编排、上下文增强、大盘环境注入、通知输出。
- `stock_analyzer.py`：规则化趋势交易分析器。
- `market_analyzer.py`：大盘环境评分和大盘复盘。
- `analyzer.py`：LLM 分析、提示词、JSON 解析、硬规则后处理。
- `data_provider/akshare_fetcher.py`：日线、实时行情、ETF、筹码分布数据。
- `search_service.py`：Tavily / SerpAPI 新闻与风险情报搜索。
- `storage.py`：SQLite 日线数据和分析上下文。

## 2.1 计划新增：热门板块驱动选股链路

后续可在现有自选股日报之外，新增一条主动选股流水线：先根据热门板块生成候选池，再用规则指标和评分体系排序，最后只对 Top 3 股票运行精细报告。

目标定位：

- 将项目从“自选股跟踪”扩展为“热点板块驱动的主动选股 + 精细分析”。
- 先用低成本规则层完成粗筛和排序，避免对大量股票直接调用 LLM。
- 只对 Top 3 候选股调用现有 `analyze_stock()` 精细流程，控制 API 成本和报告噪声。

建议流水线：

```text
热门行业/概念板块 Top 3-5
        ↓
获取板块成分股
        ↓
过滤 ST、停牌、成交额过低、涨幅过高、数据不足、一字板等不可交易标的
        ↓
批量获取日线和实时行情
        ↓
复用 StockTrendAnalyzer 计算趋势、乖离、量能、支撑评分
        ↓
叠加板块热度、流动性、相对强弱、筹码和风险过滤
        ↓
候选股综合评分排序
        ↓
Top 3 进入现有精细分析流程并生成 LLM 决策仪表盘
```

建议新增模块：

- `stock_screener.py`：热门板块选股、候选池生成、基础过滤、规则评分和排序。
- `SectorCandidate` / `ScreeningResult` 数据结构：保存板块、个股、粗筛原因、评分明细、排名。
- `main.py` 新增命令行参数：
  - `--screen-hot-sectors`：启用热门板块选股模式。
  - `--screen-top-n 3`：精细分析 Top N 股票，默认 3。
  - `--screen-sector-count 5`：用于粗筛的热门板块数量，默认 5。
  - `--screen-no-llm`：只输出规则层排序，不跑精细 LLM 报告。

选股评分建议：

| 维度 | 建议权重 | 说明 |
| --- | --- | --- |
| 板块热度 | 20 | 所属板块涨幅、成交活跃度、排名靠前程度 |
| 个股趋势 | 30 | 复用 MA5 / MA10 / MA20 / MA60 趋势评分 |
| 乖离与买点 | 20 | 低乖离、靠近 MA5 / MA10、避免追高 |
| 流动性 | 10 | 成交额、换手率，过滤交易不活跃标的 |
| 相对强弱 | 10 | 个股强于大盘、强于所属板块则加分 |
| 风险过滤 | 10 | ST、减持、业绩利空、异常放量下跌等扣分或剔除 |

基础过滤建议：

- 剔除 ST、退市风险、停牌股票。
- 剔除成交额过低标的，例如近 5 日平均成交额低于配置阈值。
- 剔除上市时间过短、历史数据不足 60 日的股票。
- 剔除当日涨幅过高且 MA5 乖离超过纪律线的股票。
- 剔除一字涨停、流动性无法正常介入的股票。
- 对连续大涨、获利盘过高且筹码分散的股票降权。

输出报告建议：

1. 热门板块概览：列出进入筛选的 Top 板块及其热度依据。
2. 候选池统计：原始成分股数量、过滤数量、有效候选数量。
3. 排序表：展示候选股代码、名称、所属板块、综合分、趋势分、乖离率、量比、成交额、过滤原因。
4. Top 3 精细报告：复用现有个股决策仪表盘。
5. 未入选原因：对接近 Top 3 但被剔除或降权的股票给出简短原因。

该能力应与现有自选股日报解耦：自选股模式负责“跟踪已有关注标的”，热门板块选股模式负责“发现新增候选标的”。

## 3. 当前金融策略逻辑

### 3.1 基础交易理念

项目内置的交易理念是：

- 严进策略：不追高，优先等待低风险买点。
- 趋势交易：优先选择 MA5 > MA10 > MA20 的多头排列。
- 效率优先：关注筹码集中、获利结构健康的股票。
- 买点偏好：趋势中缩量回踩 MA5 / MA10。
- 风控优先：大盘弱势、空头排列、明显乖离、重大利空时降低仓位或观望。

这套体系适合短中线自选股复盘，不是价值投资框架，也不是全市场高频筛选框架。

### 3.2 趋势判断

规则层计算 MA5、MA10、MA20、MA60。

短期趋势分类：

- 强势多头：MA5 > MA10 > MA20，且均线间距扩大。
- 多头排列：MA5 > MA10 > MA20。
- 弱势多头：MA5 > MA10，但 MA10 <= MA20。
- 盘整：均线缠绕或方向不清。
- 弱势空头：MA5 < MA10，但 MA10 >= MA20。
- 空头排列：MA5 < MA10 < MA20。
- 强势空头：MA5 < MA10 < MA20，且均线间距扩大。

中期趋势通过 MA60 过滤：

- 现价低于下行 MA60：中期趋势压制，限制买入类建议。
- 现价站上上行 MA60：中期趋势向好。
- 现价站上 MA60 但 MA60 未上行：中期趋势尚未确认。
- 现价低于 MA60：中期偏弱。

作用：避免只看 MA5 / MA10 / MA20 时，把下降趋势中的短期反抽误判为买点。

### 3.3 乖离率与不追高纪律

MA5 乖离率公式：

```text
bias_ma5 = (current_price - ma5) / ma5 * 100
```

当前规则：

- 价格略低于 MA5 且跌幅不深：高分，视为回踩买点。
- 价格接近 MA5：高分，视为较优介入区。
- 价格高于 MA5 但乖离 < 5%：可小仓或谨慎介入。
- 价格高于 MA5 且乖离 >= 5%：严禁追高，买入类建议被压制。

该纪律在两层执行：

1. `StockTrendAnalyzer` 评分时降低乖离过高股票的分数。
2. LLM 输出后，`_apply_hard_rules` 会强制将违反乖离规则的买入类建议改为观望，并压低评分。

这是当前系统最核心、最稳定的交易纪律之一。

### 3.4 量能逻辑

系统使用当日成交量与过去 5 日均量的比值衡量量能：

```text
volume_ratio_5d = today_volume / previous_5d_avg_volume
```

当前阈值：

- volume_ratio_5d >= 1.5：放量。
- volume_ratio_5d <= 0.7：缩量。
- 其他：量能正常。

结合价格变化后得到：

- 放量上涨：多头力量较强。
- 放量下跌：风险较高。
- 缩量上涨：上攻动能不足。
- 缩量回调：偏好形态，可能代表抛压减轻或洗盘。

当前评分偏好缩量回调，其次是放量上涨，最不偏好放量下跌。

### 3.5 支撑压力与买点

支撑判断：

- 现价距离 MA5 不超过 2%，且现价在 MA5 上方：认为 MA5 支撑有效。
- 现价距离 MA10 不超过 2%，且现价在 MA10 上方：认为 MA10 支撑有效。
- 现价在 MA20 上方：MA20 作为重要支撑位。

压力判断：

- 取近 20 日高点作为主要压力参考。

当前策略偏好：

- 多头排列中的 MA5 / MA10 回踩。
- 缩量回调后仍守住均线。
- 不偏好远离均线后的追涨。

### 3.6 筹码结构

筹码数据包括：

- 获利比例。
- 平均成本。
- 90% 筹码成本区间和集中度。
- 70% 筹码成本区间和集中度。

当前判断：

- 获利比例 > 90%：获利盘极高，警惕兑现。
- 获利比例 70%-90%：获利盘较高。
- 90% 集中度 < 8%：筹码高度集中。
- 90% 集中度 < 15%：筹码较集中。
- 90% 集中度 > 25%：筹码较分散。

硬规则：

- 如果获利比例 >= 90%，且 90% 筹码集中度 >= 15%，则提示获利盘兑现风险，并限制高置信度和高评分。

注意：ETF / 指数没有筹码分布数据，系统会跳过该部分。

### 3.7 大盘环境过滤

大盘环境从以下维度打分：

- 主要指数平均涨跌幅。
- 上涨 / 下跌家数比例。
- 涨停 / 跌停数量差。
- 两市成交额。
- 北向资金净流入。
- 领涨 / 领跌行业板块。

市场状态：

- >= 75：强势，风险低。
- >= 60：偏强，风险中。
- >= 45：震荡，风险中。
- >= 30：偏弱，风险高。
- < 30：极弱，风险高。

风控纪律：

- 大盘偏弱时，不给高置信度买入。
- 大盘极弱时，买入类建议优先改为观望。
- 极弱环境下，除非个股非常强势，否则优先控制仓位。

### 3.8 新闻与风险情报

搜索维度：

1. 最新消息。
2. 风险排查：减持、处罚、利空、风险。
3. 业绩预期：年报预告、业绩预告、业绩快报。

LLM 重点提取：

- 减持公告。
- 业绩预亏或大幅下滑。
- 监管处罚或立案调查。
- 行业政策利空。
- 大额解禁。
- 业绩、合同、订单、政策等正向催化。

该模块用于补充技术面盲区，但目前仍依赖搜索引擎摘要，不等同于权威公告源。

### 3.9 LLM 决策仪表盘

LLM 输出目标是 JSON 决策仪表盘，包含：

- 核心结论。
- 趋势预测。
- 操作建议。
- 置信度。
- 技术面透视。
- 量能分析。
- 筹码结构。
- 舆情情报。
- 风险警报。
- 作战计划。
- 买点、止损、目标价。
- 仓位建议。
- 行动检查清单。

重要原则：

- LLM 用于综合表达和解释。
- 规则层负责不可违反的交易纪律。
- LLM 输出后必须经过硬规则修正。

## 4. 当前评分体系

`StockTrendAnalyzer` 当前评分大致由四部分组成：

| 维度 | 权重 | 核心逻辑 |
| --- | --- | --- |
| 趋势 | 40 | 多头排列得分高，空头排列得分低 |
| 乖离率 | 30 | 越接近 MA5 得分越高，超过 5% 大幅扣分 |
| 量能 | 20 | 缩量回调最高，放量下跌最低 |
| 支撑 | 10 | MA5 / MA10 支撑有效则加分 |

信号映射：

- score >= 80 且趋势为强势多头或多头排列：强烈买入。
- score >= 65 且趋势至少为弱势多头：买入。
- score >= 50：持有。
- score >= 35：观望。
- 空头排列且低分：强烈卖出。
- 其他：卖出。

额外约束：

- MA5 乖离率 >= 5%：分数最高限制在 59。
- 中期趋势压制：分数最高限制在 60。
- 空头排列：禁止买入类建议。
- 大盘偏弱 / 极弱：限制买入置信度和分数。
- 获利盘过高且筹码分散：提示兑现风险并限制评分。

## 5. 当前优势

### 5.1 交易纪律清晰

系统明确知道自己偏好的交易形态：多头趋势、低乖离、缩量回踩、均线支撑。相比直接让 LLM 泛化分析，纪律更稳定。

### 5.2 规则层与 LLM 分工合理

规则层处理硬指标和风控边界，LLM 处理解释、结构化报告和新闻摘要。这降低了 LLM 幻觉或追涨表达对最终建议的影响。

### 5.3 风控意识较强

系统已经覆盖：

- 不追高。
- 空头不买。
- 弱市降仓。
- 获利盘过高预警。
- 重大利空搜索。
- MA60 中期趋势过滤。

### 5.4 多维度输入

当前输入不仅有 OHLCV，还包含实时行情、量比、换手率、筹码、大盘、板块、新闻，适合日度复盘场景。

### 5.5 更适合自选股深度跟踪

当前系统聚焦自选股，而不是全市场盲筛。该定位合理，因为 LLM、搜索、筹码和日报输出更适合少量股票的持续跟踪。

## 6. 当前不足

### 6.1 缺少回测验证

当前阈值大多来自经验，包括：

- MA5 乖离率 5%。
- MA 支撑容忍度 2%。
- 缩量阈值 0.7。
- 放量阈值 1.5。
- 买入阈值 65 / 80。
- 强势趋势间距 5%。

这些规则合理但没有数据证明，无法判断其在不同年份、行业、波动率环境下的真实胜率和收益回撤比。

### 6.2 固定阈值不适配不同波动率

不同股票波动特征不同：

- 银行、红利类股票 3% 乖离可能已经偏高。
- 科技成长股 5%-7% 乖离可能仍处于趋势内。
- ETF 与普通个股不应共用完全一致的参数。

当前固定阈值可能导致部分高波动品种过早观望，低波动品种又过度宽松。

### 6.3 买入信号缺少盈亏比

当前系统有支撑位和压力位，但没有强制计算：

- 潜在收益空间。
- 潜在止损空间。
- 盈亏比。
- 期望收益。

因此可能出现技术评分较高，但上方空间有限、下方风险较大的买入建议。

### 6.4 支撑确认偏静态

当前支撑判断只看现价是否靠近 MA5 / MA10 且站在均线上方，缺少 K 线级确认：

- 当日最低价是否触及均线。
- 收盘是否重新站上均线。
- 是否出现下影线。
- 是否缩量。
- 是否连续数日守住支撑。

因此可能把“靠近均线但仍在下跌”的情况误判为支撑有效。

### 6.5 对突破和主升浪识别较弱

当前系统更偏好回踩买点，对以下形态识别不足：

- 平台突破。
- 箱体上沿突破。
- 放量突破前高。
- 均线粘合后向上发散。
- 涨停后缩量整理。
- 龙头主升浪加速。

因此可能更稳健，但会错过部分强趋势启动或加速机会。

### 6.6 缺少行业和相对强弱

当前系统有大盘和板块涨跌榜，但没有建立“个股 - 所属行业 - 大盘”的相对强弱链条。

缺失判断：

- 个股是否强于大盘。
- 个股是否强于所属行业。
- 所属行业是否强于大盘。
- 个股是否为板块内强势标的。

这会影响短中线信号质量。

### 6.7 基本面数据结构化不足

当前基本面主要依赖 PE、PB、市值和新闻摘要，缺少：

- 营收增速。
- 净利润增速。
- ROE。
- 毛利率。
- 现金流。
- 负债率。
- 估值分位。
- 分红。
- 机构持仓。
- 财报同比 / 环比变化。

因此 LLM 的基本面分析容易依赖搜索摘要和通用常识，稳定性不足。

### 6.8 LLM 可能产生伪精确点位

当前 prompt 要求 LLM 给出具体买点、止损和目标价。若这些点位不是规则层计算，而是模型生成，就可能出现“看起来精确但依据不足”的价格。

更稳健的做法是：规则层计算点位，LLM 只解释点位来源。

### 6.9 仓位管理不足

当前仓位建议主要依赖 LLM 表述，没有明确公式化仓位模型。

缺少：

- 分数到仓位的映射。
- 大盘环境对仓位的折扣。
- 波动率对仓位的折扣。
- 单票最大亏损约束。
- 多只股票同时触发信号时的组合风险管理。

## 7. 优化路线

### 7.1 优先级 P0：建立回测模块（MVP 已完成）

目标：验证当前规则是否有效。

最小回测范围：

- 读取历史日线数据。
- 每日运行当前 `StockTrendAnalyzer`。
- 记录买入、观望、卖出信号。
- 用固定持有期或规则止损止盈模拟交易。
- 输出胜率、平均收益、最大回撤、盈亏比、交易次数。

建议验证问题：

- score >= 65 后 5 / 10 / 20 日收益是否为正。
- MA5 乖离率 5% 是否合理。
- 缩量回调是否真的优于放量上涨。
- 大盘偏弱过滤是否降低回撤。
- MA60 过滤是否减少无效反弹。

实现状态：

- 新增 `backtester.py`，提供离线规则层回测能力，不依赖 LLM、搜索、通知或实时行情。
- `TrendBacktester` 会逐日复用 `StockTrendAnalyzer`，在买入类信号出现后按次日开盘入场。
- 支持固定持有期回测，也支持使用规则层 `stop_loss` / `take_profit` 做提前退出。
- 输出 `signals_evaluated`、`buy_signal_days`、`trade_count`、`win_rate`、`average_return_pct`、`max_drawdown_pct`、`profit_loss_ratio` 等最小验证指标。
- 支持 5 / 10 / 20 日多周期回测，用于验证当前阈值在不同持有期下的表现。
- 已增加 `test_backtester.py` 覆盖交易记录、止损退出和多周期回测。

### 7.2 优先级 P0：规则层生成交易点位（MVP 已完成）

已将以下点位从 LLM 自由生成改为规则层计算：

- ideal_buy：理想买入位，优先 MA5 / MA10 或回踩支撑。
- secondary_buy：次优买入位，通常参考 MA10 / MA20。
- stop_loss：止损位，当前参考 MA20 与近 20 日低点，ATR 留待后续波动率版本。
- take_profit：目标位，参考近 20 日高点 / 压力位；无有效压力位时按规则盈亏比生成。
- risk_reward_ratio：规则层计算收益空间与止损空间的比例。
- invalidation_condition：信号失效条件。

实现状态：

- `StockTrendAnalyzer` 已输出规则层交易计划。
- `main.py` 已将规则点位注入 LLM 上下文。
- `analyzer.py` 已在提示词和硬规则后处理中要求 LLM 只解释规则点位，不自行创造点位。
- 已增加 `test_stock_analyzer.py` 覆盖点位生成、低盈亏比降级和不追高纪律。

LLM 只负责解释这些点位，不自行创造点位。

### 7.3 优先级 P0：加入盈亏比过滤（MVP 已完成）

计算：

```text
risk = entry_price - stop_loss
reward = target_price - entry_price
risk_reward_ratio = reward / risk
```

建议规则：

- risk_reward_ratio < 1.2：不买，等待更优点位。
- 1.2 <= risk_reward_ratio < 1.8：低仓位试探。
- risk_reward_ratio >= 1.8：允许正常仓位。

如果下方止损空间过大或上方压力太近，即使技术评分高，也应降级为观望。

实现状态：

- `StockTrendAnalyzer` 已在评分后应用盈亏比过滤。
- risk_reward_ratio < 1.2 时，评分限制到观望区间并等待更优点位。
- 1.2 <= risk_reward_ratio < 1.8 时，限制为低仓试探。
- `OpenAIAnalyzer._apply_hard_rules` 会在 LLM 输出后再次执行盈亏比硬规则，防止最终报告给出违反纪律的买入建议。

### 7.4 优先级 P1：波动率自适应阈值

引入 ATR 或近 20 日收益波动率。

可替代固定参数：

```text
bias_threshold = min(5%, 1.2 * atr_pct)
support_tolerance = max(1%, 0.5 * atr_pct)
stop_loss = entry_price - 1.5 * ATR
```

目标：让低波动股票更严格，高波动股票更适配自身节奏。

### 7.5 优先级 P1：K 线级支撑确认

新增支撑确认条件：

- low <= MA5 且 close >= MA5：回踩 MA5 后收回。
- low <= MA10 且 close >= MA10：回踩 MA10 后收回。
- close > open：阳线确认。
- 下影线比例较高：抛压被承接。
- 连续 2-3 日未跌破关键均线：支撑稳定。
- 跌破 MA20：买入信号失效或降级。

### 7.6 优先级 P1：相对强弱 RS 指标

增加：

- 个股近 20 日收益。
- 大盘指数近 20 日收益。
- 个股相对大盘强弱。
- 所属行业近 20 日收益。
- 个股相对行业强弱。

建议规则：

- 个股强于大盘：加分。
- 所属行业强于大盘：加分。
- 个股弱于所属行业：降分。
- 大盘弱但个股和行业都强：允许低仓试探。

### 7.7 优先级 P1：仓位管理模型

建议初版公式：

```text
base_position:
- score >= 85: 30%
- score >= 75: 20%
- score >= 65: 10%
- score < 65: 0%

market_multiplier:
- 强势: 1.0
- 偏强: 0.8
- 震荡: 0.5
- 偏弱: 0.3
- 极弱: 0.0

risk_reward_multiplier:
- rr >= 2.0: 1.0
- 1.5 <= rr < 2.0: 0.7
- rr < 1.5: 0.0

final_position = base_position * market_multiplier * risk_reward_multiplier
```

同时加入单票最大亏损约束：

```text
position_size <= account_risk_budget / abs(entry_price - stop_loss)
```

### 7.8 优先级 P1：热门板块驱动选股能力

新增独立选股流水线，先利用热门板块做粗筛，再利用指标和评分体系排序，最后只对 Top 3 股票运行精细 LLM 报告。

MVP 范围：

1. 获取当日行业 / 概念热门板块 Top 3-5。
2. 获取板块成分股并合并去重。
3. 做基础过滤：ST、停牌、成交额过低、涨幅过高、历史数据不足、一字板等。
4. 对候选股批量获取日线和实时行情。
5. 复用 `StockTrendAnalyzer` 计算趋势评分。
6. 叠加板块热度、流动性、相对强弱和风险过滤，形成综合分。
7. 取 Top 3 进入现有 `analyze_stock()` 精细分析链路。
8. 输出“热门板块选股报告 + Top 3 个股决策仪表盘”。

建议先实现规则层排序，不急于让 LLM 参与候选池筛选。LLM 只处理最终 Top 3，避免成本过高和解释噪声过大。

### 7.9 优先级 P2：公告和财报数据源

建议接入或扩展：

- 巨潮资讯 / CNINFO 公告。
- 交易所公告。
- Tushare 财务指标。
- 业绩预告。
- 股东减持。
- 限售解禁。
- 机构调研。
- 龙虎榜。

权重建议：

- 官方公告 > 财务结构化数据 > 主流财经媒体 > 搜索摘要。

风险排查不应长期依赖搜索引擎摘要。

### 7.10 优先级 P2：区分股票、ETF、指数策略

不同品种应使用不同策略权重。

普通股票：

- 筹码、业绩、减持、PE/PB、股东行为权重更高。

行业 ETF：

- 行业趋势、板块资金流、成分股表现、政策催化权重更高。

宽基 ETF / 指数：

- 大盘成交额、市场宽度、宏观流动性、估值分位权重更高。

### 7.11 优先级 P2：识别突破与趋势加速形态

补充形态：

- 近 20 日新高。
- 放量突破平台。
- 均线粘合后向上发散。
- 涨停后缩量整理。
- 回踩突破位不破。
- 板块共振突破。

需要注意：突破策略应和当前回踩策略分开评分，避免把“追高”与“有效突破”混为一谈。

## 8. 推荐迭代顺序

建议按以下顺序优化：

1. 增加回测模块，先验证当前策略。（MVP 已完成）
2. 将买点、止损、目标位改为规则层计算。
3. 加入盈亏比过滤。
4. 加入明确仓位模型。
5. 用 ATR / 波动率替代部分固定阈值。
6. 升级支撑确认逻辑。
7. 增加相对强弱和行业映射。
8. 增加热门板块驱动选股能力。
9. 接入公告、财报和解禁数据。
10. 区分普通股票、ETF、指数策略。
11. 再考虑突破、主升浪、板块龙头等进攻型策略。

## 9. 后续设计原则

后续优化时建议坚持：

- 规则层负责计算和约束，LLM 负责解释和报告。
- 任何买入类建议必须能追溯到具体规则。
- 点位、止损、仓位不要由 LLM 自由生成。
- 新增指标必须能被回测验证。
- 每个信号都要有失效条件。
- 弱市优先控制风险，而不是强行寻找机会。
- 不同品种使用不同参数，不要让 ETF、低波动股票、高波动成长股共用全部阈值。
- 文档、规则、提示词和测试应同步更新。
